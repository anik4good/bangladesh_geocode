GeoAPIBD API Documentation

This document provides an overview of the APIs implemented in the controllers package. Each endpoint is described with its HTTP method, URL, input parameters, and the response format.

1. Hello Endpoint

Description: Provides basic information about the API and available endpoints.

HTTP Method: GET

URL: /api/hello

Response:

{
    "Status": true,
    "Name": "Welcome to GeoAPIBD. Powered by GO, Developed by Anik",
    "Divisions": "<SERVER_URL>/api/divisions",
    "Districts": "<SERVER_URL>/api/districts",
    "District": "<SERVER_URL>/api/division/rangpur",
    "Upozilla": "<SERVER_URL>/api/division/rangpur/panchagarh",
    "Unions": "<SERVER_URL>/api/division/rangpur/panchagarh/debiganj"
}

2. Get All Divisions

Description: Retrieves all divisions from the database.

HTTP Method: GET

URL: /api/divisions

Response (on success):

{
    "Status": true,
    "Message": "ALL Divisions",
    "Time": "2025-01-25 10:30:00",
    "Data": [
        {
            "ID": 1,
            "Name": "Division Name",
            "...": "Other Fields"
        }
    ]
}

Response (on failure):

{
    "Status": true,
    "Message": "No Data Found",
    "Time": "2025-01-25 10:30:00"
}

3. Get Division by ID

Description: Fetches a specific division by its ID.

HTTP Method: GET

URL: /api/divisions/:id

Path Parameter:

  • id (string): ID of the division.

Response (on success):

{
    "ID": 1,
    "Name": "Division Name",
    "...": "Other Fields"
}

Response (on failure):

{
    "Status": true,
    "Message": "No Data Found",
    "Time": "2025-01-25 10:30:00"
}

5. Get Districts by Division Name

Description: Fetches all districts under a specific division.

HTTP Method: GET

URL: /api/divisions/:division_name/districts

Path Parameter:

  • division_name (string): Name of the division.

Response (on success):

{
    "Status": true,
    "Message": "ALL Districts Under <Division Name>",
    "Time": "2025-01-25 10:30:00",
    "Data": [
        {
            "ID": 1,
            "Name": "District Name",
            "...": "Other Fields"
        }
    ]
}

Response (on failure):

{
    "Status": true,
    "Message": "No Division Found",
    "Time": "2025-01-25 10:30:00"
}
